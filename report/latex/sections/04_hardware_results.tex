\section{硬件实现结果}

本章汇总 Vivado 综合与实现得到的 LUT、FF、DSP、BRAM、时钟频率、功耗和归一化指标。

当前主线器件已经切换为 \texttt{XCZU4EV-SFVC784-2I}，默认目标周期为 \texttt{3.333 ns}。截至目前，五个自研 RTL 架构均已完成 ZU4EV 上的 post-route 数据收集，其中 \texttt{fir\_pipe\_systolic} 是唯一在该目标周期下取得正裕量的版本，估计 \texttt{Fmax = 459.348 MHz}，同时保持最低的 \texttt{energy/sample}。

\texttt{fir\_l2\_polyphase} 与 \texttt{fir\_l3\_polyphase} 证明了并行 polyphase datapath 在大器件上的可实现性：前者达到约 \texttt{278.668 MS/s}，后者达到约 \texttt{381.388 MS/s}。不过从当前结果看，\texttt{L=3} 仍未超越高质量标量流水线的综合表现，因此 ZU4EV 平台下的英雄架构仍然是 \texttt{fir\_pipe\_systolic}。完整数据以仓库内 \texttt{data/impl\_results.csv} 为唯一真源。

本轮还额外补齐了 \texttt{board-shell scope} 的结果，即把 \texttt{PS + AXI DMA + FIR shell + bare-metal harness} 一并纳入实现统计。在这一口径下，\texttt{zu4ev\_fir\_pipe\_systolic\_top} 与 \texttt{zu4ev\_fir\_vendor\_top} 都完成了 bitstream、`.xsa` 与板上验证。前者估计 \texttt{Fmax = 347.826 MHz}、\texttt{LUT = 20253}、\texttt{energy/sample = 8.542 nJ}；后者估计 \texttt{Fmax = 347.102 MHz}、\texttt{LUT = 8856}、\texttt{energy/sample = 8.243 nJ}。这说明在最终系统层面对照中，vendor FIR IP 的资源与能效略优，而自研系统壳保持了几乎相同的频率水平。

为了让这些结果更具有解释力，本轮还补充了归一化效率指标。以 kernel scope 为例，\texttt{fir\_pipe\_systolic} 的 \texttt{throughput/DSP} 达到约 \texttt{3.480}，显著高于 \texttt{fir\_symm\_base} 与 \texttt{fir\_l2\_polyphase}；而 \texttt{fir\_l2\_polyphase} 的 \texttt{throughput/kLUT} 很高，说明它在 LUT 利用效率上非常强，但 DSP 代价偏大。系统壳层面则出现不同排序：\texttt{vendor} 与 \texttt{custom} 的频率几乎一致，但 vendor 的 \texttt{throughput/kLUT} 更高、\texttt{energy/sample} 更低。

当前功耗口径来自 routed \texttt{report\_power} 的 vectorless 估计，置信度为 \texttt{Medium}，因此应把这些数字理解为相对比较数据而不是最终 sign-off 功耗。对于本项目来说，更有价值的是功耗分层：在 board-shell scope 下，\texttt{PS8} 约固定在 \texttt{2.228 W}，真正体现 FIR 结构差异的是 \texttt{fir\_shell\_0}，自研系统壳约 \texttt{0.238 W}，vendor 系统壳约 \texttt{0.132 W}。

时序分析同样给出了重要结论。当前 board-shell 最坏路径已经不是 FIR 核内部的 DSP 乘加，而是 FIR 输出经过 \texttt{round/saturate} 再写入输出 FIFO BRAM 的路径。自研系统壳该路径的 \texttt{Data Path Delay = 2.859 ns}，其中逻辑延迟约 \texttt{1.348 ns}、布线延迟约 \texttt{1.511 ns}，route 占比超过一半。这说明当 FIR 核本体已经高度流水化以后，系统层性能瓶颈会自然推进到输出边界与局部互连，而不再停留在乘法器本身。
