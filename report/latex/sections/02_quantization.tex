\section{Quantization Analysis}

This chapter compares the frequency response of the floating-point and fixed-point filters and analyzes the impact of coefficient quantization, internal overflow risk, and the output-rounding strategy.

The current project did not treat ``100 taps'' as the final answer directly. Instead, it preserved the two baselines \texttt{baseline\_taps100} and \texttt{baseline\_order100}. Under \texttt{firpm}, both only achieve stopband attenuation at roughly the \texttt{40 dB} level, so the final solution that truly satisfies \texttt{Ast >= 80 dB} had to be raised to \texttt{order = 260 / 261 taps}. This step is important because it shows that the current challenge is not just coefficient quantization; the floating-point design itself must first have enough stopband margin.

The fixed-point sweep shows that, under the current default \texttt{Wout = 16}, \texttt{Wcoef = 20} is the first point that stably crosses the \texttt{80 dB} stopband threshold: \texttt{Wcoef = 16} gives only about \texttt{67.7 dB}, \texttt{Wcoef = 18} gives about \texttt{77.2 dB}, and \texttt{Wcoef = 20} reaches \texttt{81.3994 dB}. Increasing further to \texttt{22} or \texttt{24} bits still improves the stopband slightly, but the gain is already clearly marginal, so the project finally settles on \texttt{Q1.15 + Wcoef20 + Wout16 + Wacc46}.

The internal bit width uses the conservative formula
\[
W_{\mathrm{acc}} = W_{\mathrm{in}} + W_{\mathrm{coef}} + \lceil \log_2 N_{\mathrm{uniq}} \rceil + 2
\]
For the current \texttt{261 taps} design, where symmetry folding gives \texttt{N\_uniq = 131}, we have
\[
W_{\mathrm{acc}} = 16 + 20 + \lceil \log_2 131 \rceil + 2 = 46 .
\]
Using the sum of absolute quantized coefficients, we obtain the conservative integer-domain upper bound
\[
|y|_{\max} \le |x|_{\max} \sum_k |h_q[k]| = 32767 \times 1{,}153{,}458 = 37{,}795{,}358{,}286 ,
\]
which only requires \texttt{37} signed bits to contain, while the current \texttt{46-bit} accumulator still leaves \texttt{9-bit} headroom. The observed \texttt{max\_abs\_acc} from the actual sweep is lower still, so the current \texttt{overflow\_count = 0} is not accidental; it is supported by the chosen bit-width budget.

In implementation, the project insists on full-precision internal accumulation and performs a unified \texttt{round-to-nearest + saturate} operation only at the final output boundary. The value of this choice is that the golden vectors, RTL, vendor baseline, and on-board bare-metal harness all share the same final-stage quantization rule, preserving a truly bit-true comparison frame.
