\section{分析与总结}

本章总结各架构的性能、效率和适用场景，给出本项目的 \textit{performance hero} 与 \textit{efficiency hero}。

在当前 ZU4EV 主线下，\textit{performance hero} 与 \textit{efficiency hero} 都由 \texttt{fir\_pipe\_systolic} 获得。它不仅取得最高吞吐，而且是当前唯一在 \texttt{3.333 ns} 目标周期下闭合时序的自研架构。这说明对于本项目这一类 261 taps、线性相位、严格 bit-true 的 FIR 设计，高质量的对称折叠加 systolic 流水线仍然是非常强的工程解。

\texttt{L=2} 与 \texttt{L=3} 的研究价值依然成立。\texttt{L=2} 展示了真正 polyphase + symmetry 结构如何进入统一验证和实现链；\texttt{L=3} 则证明共享 \texttt{L3 FFA core} 的压缩路线在 ZU4EV 上已经具备可展示的高吞吐潜力。与此同时，\texttt{PS + PL + Bare-metal Vitis} 的系统级流程也已经闭环：\texttt{fir\_pipe\_systolic} 与 \texttt{vendor FIR IP} 都完成了自动下载、自动串口抓取和板上用例验证，且板上 \texttt{mismatch\_count = 0}。

最终项目结论需要按两个口径诚实呈现。若讨论自研研究型 RTL 矩阵，冠军仍然是 \texttt{fir\_pipe\_systolic}；若讨论最终系统壳对照，\texttt{vendor FIR IP} 在保持几乎相同频率的前提下，取得了更低的 LUT、FF 和 \texttt{energy/sample}。因此，本项目的价值不在于“强行证明自研一定全面胜出”，而在于：我们已经在统一规格、统一固定点、统一向量、统一板测链路下，把自研架构与工业基线放到了同一张可复现的对照表里。

本轮还补齐了两个对最终结论很重要的分析维度。第一是稳定性：\texttt{fir\_pipe\_systolic} 与 \texttt{vendor FIR IP} 都已经在当前正式 \texttt{8-case} 板测套件下完成最近 \texttt{3} 次窗口重复运行，且全部 \texttt{8/8} case 通过、\texttt{mismatch\_sum = 0}。这说明当前自动闭环并不是“一次跑通”，而是已经具备稳定重复运行能力。第二是方法论透明度：当前仓库不仅保留最终数字，还把方法口径、功耗分层、关键路径分解、位宽推导和稳定性统计都沉淀成了 CSV/JSON 真源，后续无论是 GitHub Pages 还是课程报告，都可以直接从这些真源生成一致的图表和表格。
