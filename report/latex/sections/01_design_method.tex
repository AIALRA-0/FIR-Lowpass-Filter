\section{Design Method}

This chapter explains the course specification, the MATLAB normalization convention, the rationale for keeping dual baselines, and the design flow that led to the final filter meeting the specification.

\subsection{Specification Freeze}

The project specification uses \texttt{spec/spec.json} in the repository as the single source of truth, and all MATLAB, RTL, Vivado, and documentation artifacts read their core constraints from that file.

\subsection{Design Route}

The main design method uses Parks-McClellan equiripple design, with least-squares and Kaiser window kept as comparison methods.
